# Dynamic Tour – The Ultimate Travel Booking Experience

![Hero](./screenshots/wbsender_landing.jpg)

**Dynamic Tour** is your all-in-one travel booking platform designed to make planning and managing trips effortless. From exploring bucket-list destinations to managing bookings and payments — everything is built to be seamless, fast, and modern.

Whether you're chasing waterfalls or northern lights, Dynamic Tour helps you get there in style. 🌍✈️

---

## ✨ Features

- 🌐 **Explore Curated Tours** – Browse stunning tours with rich visuals and in-depth info
- 🧠 **Smart Filtering** – Filter tours by price, location, duration, or rating
- ⚡ **Instant Booking** – Real-time availability with one-click tour booking
- 🧾 **User Dashboard** – Manage bookings, reviews, and personal info with ease
- 💳 **Secure Payments** – Stripe-powered checkout for a smooth and secure experience
- 🛠️ **Admin Panel** – Manage tours, users, and reviews (admin only)

## 🛠️ Tech Stack

### Frontend

- **Next.js** (App Router)
- **Tailwind CSS**
- **TanStack Query** (React Query)

### Backend

- **Node.js** with **Express.js**
- REST API

### Database

- **MongoDB** with **Mongoose**

### Auth & Validation

- **JWT** (Authentication)
- **Zod** (Validation)

### Payments

- **Stripe API** (with webhook support)

---

## 🔐 Role-Based Access

| Role                | Permissions                                             |
| ------------------- | ------------------------------------------------------- |
| Public Users        | View and explore tours                                  |
| Authenticated Users | Book tours, leave reviews, manage profile               |
| Admin               | Full access to manage tours, users, orders, and reviews |

---

## 🧠 Key Learnings

- Modular and reusable components built with clean architecture
- Auth flow with protected routes and role-based access (JWT)
- Implemented Stripe payments + webhooks for booking updates
- Lazy-loading images, SSR rendering, and query caching for speed
- Dashboard segregation for users vs admins using clean layouts
